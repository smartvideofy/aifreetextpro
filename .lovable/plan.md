# Reality check on Option B

I owe you a correction — I was wrong to offer Option B as if it were a code fix I could do.

**Lovable Cloud can't be disconnected from this project.** Per the platform docs: once Cloud is enabled on a project, it stays enabled. Reverting to a pre-Cloud version doesn't remove it either. The only Cloud toggle available is under **Connectors → Lovable Cloud → Disable Cloud**, and that only affects *future* projects you create — not this one.

That means:

- `src/integrations/supabase/client.ts` will keep being regenerated by the platform even if I delete it.
- I can't remove Cloud from code, and neither can you from this project's settings.
- The build error stays until `@supabase/supabase-js` is installed.

## What I recommend

Go with Option A after all — install the dep. It is genuinely inert for your marketing site:

- Nothing in `src/` imports the client (I grep-verified: only `client.ts` itself references it).
- Vite tree-shakes unused modules, so the shipped bundle does not grow.
- Zero runtime cost, zero SEO cost, unblocks the build.
